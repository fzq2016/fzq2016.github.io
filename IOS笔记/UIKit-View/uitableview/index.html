UIKit-View

UITableView

  1. 父类是UIScrollView,可以滚动，苹果对其进行了性能优化，性能极佳
  2. UITableView需要一个数据源(dataSource)来显示数据，数据源一般是控制器(主要是UITableViewController)
  3. 没有设置数据源的UITableView只是个空壳

UITableView的创建及样式

// UITableViewStylePlain：普通样式
self.tableView = [[UITableView alloc]initWithFrame:self.view.frame  style:UITableViewStylePlain];

// UITableViewStylePlain：分组样式
self.tableView = [[UITableView alloc]initWithFrame:self.view.frame  style:UITableViewStyleGrouped];

如何让tableView展示数据

   概况：UITableView会向数据源查询一共有多少组、多少行数据以及每一行显示什么数据等，实现步骤：

  1. 设置数据源对象

    tableView.dataSource = self;

  2. 数据源对象要遵守数据源协议

    @interface ViewController () <UITableViewDataSource>

  3. 实现数据源方法

      // 多少组数据(可选)
  - (NSInteger)numberOfSectionsInTableView:(UITableView *)tableView
  {
      return 1;
  }

  // 每一组有多少行数据
  - (NSInteger)tableView:(UITableView *)tableView numberOfRowsInSection:(NSInteger)section
  {
      return 10;
  }

  // 每一行显示什么内容
  - (UITableViewCell *)tableView:(UITableView *)tableView cellForRowAtIndexPath:(NSIndexPath *)indexPath
  {
      // 定义重用标识
      // 生成cell代码
  }

  // 返回头部高度
  -(CGFloat)tableView:(UITableView *)tableView heightForHeaderInSection:(NSInteger)section
  {
      return 50;
  }

  // 返回脚部高度
  -(CGFloat)tableView:(UITableView *)tableView heightForFooterInSection:(NSInteger)section
  {
      return 50;
  }

  // 每一组的头部
  - (NSString *)tableView:(UITableView *)tableView titleForHeaderInSection:(NSInteger)section
  {
      return @"头部";
  }

  // 每一组的尾部
  - (NSString *)tableView:(UITableView *)tableView titleForFooterInSection:(NSInteger)section
  {
      return @"尾部";
  }

UITableView的内容->cell的获取

UITableView的内容是多个定制的cell；其内容展示交由cell来管理；

  • 方案一：缓存中取出带有标记的cell

    • 考虑到循环利用，cell的获取优先考虑方案一，当方案一失效时利用另外四种方案中的任一种

    • 关于循环利用，详见UITableViewCell节中UITableViewCell的重用原理

      // 重用标识
static NSString *ID = @"cell";

// 缓存中取出cell
UITableViewCell * carCell = [tableView dequeueReusableCellWithIdentifier:ID];
  • 方案二：initWithStyle: reuseIdentifier:创建
      // 需先判断缓存中是否有cell
  if(cell == nil){
      carCell = [UITableViewCell carCellWithStyle:UITableViewCellStyleDefault reuseIdentifier:ID];
  }
  • 方案三：注册cell，cell的获取由系统管理
    [tableView registerClass:[UITableViewCell class] forCellReuseIdentifier:ID];
  • 方案四：注册nib，系统会自动加载nib中的cell
    [tableView registerNib:[UINib nibWithNibName:NSStringFromClass(XXXCell.class) bundle:nil] forCellReuseIdentifier:ID];
  • 方案五：在storyboard中的tableView中拖cell控件，修改cell的class为自定义的cell（可选，也可以使用系统的cell）,并绑定循环引用标识即可
    • 修改cell的class为XXXTgCell
    • 绑定循环利用标识

UITableView的常用属性

// 真实高度
// 设置每一行cell的统一高度
tableView.rowHeight = 100;
// 设置每一组头部的统一高度
tableView.sectionHeaderHeight = 50;
// 设置每一组尾部的统一高度
tableView.sectionFooterHeight = 50;

// 预计高度
// tableView显示时，会调用对应方法访问即将创建的所有cell、Header、Footer的高度，用于计算右侧指示器长度；
// 如果设置了预计高度，只会访问即将显示的内容的高度，其他不显示的cell、Header、Footer的高度不会访问，用预计高度计算右侧指示器长度
// 开发中建议设置预计高度，减少tableView频繁访问高度，可优化性能
tableView.estimatedRowHeight = 100;
tableView.estimatedSectionHeaderHeight = 100;
tableView.estimatedSectionFooterHeight = 100;

// 设置分隔线颜色
tableView.separatorColor = [UIColor redColor];
// 设置分隔线样式（默认是一条线，但宽度不是整个屏幕的宽度）
tableView.separatorStyle = UITableViewCellSeparatorStyleNone;//不显示分隔线；

// 设置表头控件
tableView.tableHeaderView = [[UISwitch alloc] init];
// 设置表尾控件
tableView.tableFooterView = [UIButton buttonWithType:UIButtonTypeContactAdd];

// 设置右边索引文字的颜色
tableView.sectionIndexColor = [UIColor redColor];
// 设置右边索引文字的背景色
tableView.sectionIndexBackgroundColor = [UIColor blackColor];

数据操作

  1. 添加数据

     // 修改模型
 [model insertObject:newObject atIndex:0];
 [model insertObject:newObject atIndex:0];

 // 增加指定cell到对应位置
NSArray *indexPaths = @[
                     [NSIndexPath indexPathForRow:0 inSection:0],
                     [NSIndexPath indexPathForRow:1 inSection:0]
                     ];
[tableView insertRowsAtIndexPaths:indexPaths withRowAnimation:UITableViewRowAnimationRight];

  2. 删除数据

          // 修改模型
      [model removeObjectAtIndex:0];
      [model removeObjectAtIndex:0];

      // 删除对应位置的cell
  NSArray *indexPaths = @[
                          [NSIndexPath indexPathForRow:0 inSection:0],
                          [NSIndexPath indexPathForRow:1 inSection:0]
                          ];
  [tableView deleteRowsAtIndexPaths:indexPaths withRowAnimation:UITableViewRowAnimationMiddle];

  3. 更新数据（没有添加和删除数据，仅仅是修改已经存在的数据）

    • 全局刷新方法(最常用)
        // 刷新屏幕上的所有可视的cell
  [tableView reloadData];

    • 局部刷新方法（要先修改Model中的数据再刷新）

        // 修改模型数据
  [model replaceObjectAtIndex:1 withObject:newObject];

  // 刷新指定cell
  NSArray *indexPaths = @[
                          [NSIndexPath indexPathForRow:0 inSection:0],
                          [NSIndexPath indexPathForRow:1 inSection:0]
                          ];
  [tableView relaodRowsAtIndexPaths:indexPaths withRowAnimation:UITableViewRowAnimationMiddle];

UITableView编辑状态

  • 需要实现tableView的代理方法

  • 是否允许编辑

      - (BOOL)isEditing{
      return NO;
  }

  • 左滑出现删除按钮

      // 实现了这个方法，左滑就会出现Delete按钮
  // 点击了“左滑出现的Delete按钮”会调用这个方法
  - (void)tableView:(UITableView *)tableView commitEditingStyle:(UITableViewCellEditingStyle)editingStyle forRowAtIndexPath:(NSIndexPath *)indexPath
  {
      // 删除模型数据
      [self.model removeObjectAtIndex:indexPath.row];

      // 刷新
      [tableView deleteRowsAtIndexPaths:@[indexPath] withRowAnimation:UITableViewRowAnimationLeft];
  }

  // 修改Delete按钮文字为“删除”
  - (NSString *)tableView:(UITableView *)tableView titleForDeleteConfirmationButtonForRowAtIndexPath:(NSIndexPath *)indexPath
  {
      return @"删除";
  }

  • 左滑出现N个按钮

      // 实现了这个方法，左滑就会出现功能按钮
  // 一旦左滑出现了N个按钮，tableView就进入了编辑模式, tableView.editing = YES
  - (void)tableView:(UITableView *)tableView commitEditingStyle:(UITableViewCellEditingStyle)editingStyle forRowAtIndexPath:(NSIndexPath *)indexPath
  {
      // 可不写任何代码，也有左滑进入编辑状态的功能
  }

  // 左滑cell时出现什么按钮
  - (NSArray *)tableView:(UITableView *)tableView editActionsForRowAtIndexPath:(NSIndexPath *)indexPath
  {
      // 按钮一
      UITableViewRowAction *action0 = [UITableViewRowAction rowActionWithStyle:UITableViewRowActionStyleNormal title:@"关注" handler:^(UITableViewRowAction *action, NSIndexPath *indexPath) {
          NSLog(@"点击了关注");

          // 收回左滑出现的按钮(退出编辑模式)
          tableView.editing = NO;
  }];

      // 按钮二
      UITableViewRowAction *action1 = [UITableViewRowAction rowActionWithStyle:UITableViewRowActionStyleDefault title:@"删除" handler:^(UITableViewRowAction *action, NSIndexPath *indexPath) {
      // 修改模型数据
      [self.model removeObjectAtIndex:indexPath.row];

      // 刷新cell
          [tableView deleteRowsAtIndexPaths:@[indexPath] withRowAnimation:UITableViewRowAnimationAutomatic];
      }];

      // 返回按钮
      return @[action1, action0];
  }

  • 进入编辑模式

      // 默认情况下，进入编辑模式时，左边会出现一排红色的“减号”按钮
  // self.tabelView.editing = YES; // 无动画效果
  [tableView setEditing:YES animated:YES]; // 带动画效果

  • 在编辑模式中多选

      // 编辑模式的时候可以多选
  tableView.allowsMultipleSelectionDuringEditing = YES;

  // 进入编辑模式
  [tableView setEditing:YES animated:YES];

  // 获得选中的所有行
  tableView.indexPathsForSelectedRows;
