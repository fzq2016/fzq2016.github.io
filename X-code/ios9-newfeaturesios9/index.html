ios9-NewKeyword （ios新特性-‘空’关键字）

###‘空’关键字综述
/*
    关键字目的:
    1.提高代码规范,减少程序员之间交流
    2.迎合swift,在swift必须要指明这个变量是否为空.

    关键字注意点:
    1.关键字只对方法有效果,对属性没有效果(如：_name=nil不会警告，尽管显示需要nonnull的值)
    2.关键字修饰方法参数,方法返回值
    3.关键字只能用来修饰对象,不能修饰基本数据类型（基本数据类型不能赋值为nil）
    4.容器类OC对象，get不能为空。

    NONNULL宏：
    1.NS_ASSUME_NONNULL_BEGIN和NS_ASSUME_NONNULL_END之间的所有方法（@property本质也是方法调用）,默认方法的参数与返回值都是nonnull；
    2.放在属性前后没有效果。
    3.若宏之间的方法的参数或返回值已经用空关键字修饰，不会重复使用nonnull修饰该方法参数或返回值

 */

###NS_ASSUME_NONNULL_BEGIN和NS_ASSUME_NONNULL_END的使用
NS_ASSUME_NONNULL_BEGIN

@property(nonatomic ,assign) int age;

//@property (nonatomic, strong) NSString *name;
//@property (nonatomic, strong) NSString * __nullable name;
//@property (nonatomic, strong) NSString * __nonnull icon;
@property (nonatomic, strong , null_resettable) NSString * status;

@end

NS_ASSUME_NONNULL_END


###nullable、nonnull、null_resettable、_Null_unspecified

    研究:作用,怎么使用(代码格式)

    1.nullable:可以为空
    方式一:
    @property (nonatomic, strong, nullable) NSString *name;

    方式二:
    @property (nonatomic, strong) NSString * _Nullable name;

    方式三:
    @property (nonatomic, strong) NSString * __nullable name;

    2.nonnull:不能为空
     方式一:
     @property (nonatomic, strong, nonnull) NSString *icon;

     方式二:
     @property (nonatomic, strong) NSString * _Nonnull icon;

     方式三:
     @property (nonatomic, strong) NSString * __nonnull icon;

    3.null_resettable:get返回永远有值,set方法可以为空
    方式一:
    @property (nonatomic, strong, null_resettable) NSString *status;

    注意:必须要重写get或者set方法处理为空的情况

    4._Null_unspecified:不确定是否为空；有可能get不能为空，set可以为空，也可能相反等等；开发中使用较少

iOS9新特性之泛型

泛型好处:
    1.提高代码规范,程序员一看就知道是什么
    2.编译检测,防止传入不想要的数据
    3.从集合中取出值,能直接使用点语法

泛型注意:
    泛型作用于方法调用,一般用于可变集合方法中
    系统集合类型已经声明了泛型

泛型格式:
    泛型定义:
        在类声明的外面,类型后面,<集合中元素类型>
    泛型声明:
        在声明类的时候,在类型后面<泛型名称>

泛型特性:
    1.只要发现类型不匹配,就会报警告,但不会报编译错误
    2.如果没有定义泛型,默认就是id

泛型开发使用场景:
    1.限制集合类型
    2.在声明一个类的时候,某个属性类型不确定,可以使用泛型


###自定义类泛型使用示例
/*
    类型转换

    __covariant:协变->子类转父类

    __contravariant:逆变->父类转子类

 */

// 泛型声明:
//在声明类的时候,在类型后面<泛型名称>
@interface Person<__contravariant ObjectType> : NSObject

// 语言
@property (nonatomic ,strong)  ObjectType lauguage;
@end


//泛型定义:
//在类声明的外面,类型后面,<集合中元素类型>
@property (nonatomic ,strong) NSMutableArray<NSString *> *arr;


### 协变:子类给父类赋值--泛型声明中关键字为__covariant
    //创建泛型为IOS* 的Person对象
    Person<IOS *> *p = [[Person alloc] init];

    //创建泛型为Language* 的Person对象
    Person<Language *> *p1;

    // 协变:子类给父类赋值
    p1 = p;


### 逆变:父类给子类赋值--泛型声明中关键字为__contravariant
    //创建泛型为Language* 的Person对象
    Person<Language *> *p = [[Person alloc] init];

    //创建泛型为IOS* 的Person对象
    Person<IOS *> *p1;

    //逆变:父类给子类赋值
    p1 = p;

__kindof

1. xcode4（ios6）之前：
//返回值类型是id，不会进行编译检查，容易出错
//+ (id)person;

2. xcode5（ios7）
//推出instancetype:自动识别方法调用者类型。若调用者与返回值类型不一致，会报警告。
+ (instancetype)person;

3. xcode7（ios8）:
// __kindof:修饰返回值或者参数类型,表示当前类或者子类
//优点：1.会进行编译检查；2.提高代码规范，能告知程序员返回值得类型，而不仅仅是instancetype
//+ (__kindof Person *)person;

4. 编译检查(id不会进行编译检查示例)：
//+ (id)person;//Subperson的 Person,返回值是id

NSString *p = [Subperson person];//不会警告
Subperson *p1 = [NSString string];//会警告
