iOS监听事件方法

    01 block
        1.1 作为成员变量或者作为方法的参数回调block
        1.2 高级开发人员多用block
        1.3 要注意block对self、_成员属性的强引用造成的循环引用
        1.4 详见“block”

    02 UIControl及其子类的通过下面方法监听：
- (void)addTarget:(nullable id)target action:(SEL)action forControlEvents:(UIControlEvents)controlEvents;

    03 代理
        2.1 使用范围
            a 1对1：1个对象只能设置一个代理(假设这个对象只有1个代理属性)
            b 1个对象能成为多个对象的代理
        2.2 比`通知`规范
        2.3 建议使用`代理`多于`通知`

    04 通知
        3.1 任何对象之间都可以传递消息
        3.2 使用范围
            a 1个对象可以发通知给N个对象
            b 1个对象可以接受N个对象发出的通知
        3.3 必须得保证通知的名字在发出和监听时是一致的
        3.4 详见"NSNotificationCenter"\"NSNotification"章节

    05 KVO
        4.1 仅仅是能监听对象属性的改变（灵活度不如通知和代理）
        4.2 本质是为每一个监听对象添加一个观察者对象
        4.3 详见"KVO"

    06 利用内部的某些机制
        6.1 比如重写UITextField的`becomeFirstResponder`和`resignFirstResponder`来监听UITextField的获得焦点和失去焦点事件

/**
 *  调用时刻 : 成为第一响应者(开始编辑\弹出键盘\获得焦点)
 */
- (BOOL)becomeFirstResponder
{
    // 设置占位文字颜色
    self.zq_placeholderColor = [UIColor whiteColor];

    return [super becomeFirstResponder];
}

/**
 *  调用时刻 : 不做第一响应者(结束编辑\退出键盘\失去焦点)
 */
- (BOOL)resignFirstResponder
{
    // 恢复默认占位文字颜色
    self.zq_placeholderColor = [UIColor grayColor];

    return [super resignFirstResponder];;
}
